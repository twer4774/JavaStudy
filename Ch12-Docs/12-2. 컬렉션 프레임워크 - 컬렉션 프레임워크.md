# 12-2. 컬렉션 프레임워크 - 컬렉션 프레임워크

## 컬렉션 프레임워크란?

- 자바에서 미리 구현해 놓은 자료구조입니다.

## Collection 인터페이스

- Collection 인터페이스 하위에는 List 인터페이스와 Set 인터페이스가 있습니다.
  - List 인터페이스 :  순차적인 자료 관리
    - ArrayList, Vector, LinkedList, Stack, Queue 등이 있습니다.
  - Set 인터페이스 : 순서가 없으며, 중복을 허용하지 않는 자료 관리
    - HashSet, TreeSet 등이 있습니다.
- Collection에서 자주 사용되는 메소드
  - boolean add(E e)
  - void clear()
  - Iterator\<E> iterator : 순환할 반복자를 반환합니다.
  - boolean remove(Object o) : 해당 인스턴스가 존재하면 제거합니다.
  - int size()

## Map 인터페이스

- 키와 값의 쌍으로 표현하며 키는 중복될 수 없고, 값은 중복을 허용합니다.
  - HashTable, HashMap, TreeMap 등이 있습니다.
- 주요 메소드
  - V put(K key, V value)
  - V get(K key)
  - boolean isEmpty()
  - boolean containsKey(Object key) : 해당 key가 있는지의 여부
  - boolean containsKey(Object value)
  - Set keySet() : key 집합을 Set으로 반환
  - Collection values() : value를 Collection으로 반환
  - V remove(key) : key가 있는 경우 삭제
  - boolean remove(Object key, Object value) : key에 해당하는 value가 매개변수와 일치할 때 삭제

## 실습 - 회원 관리 프로그램

```java
package Collection;

public class Member {
    private int memberId;
    private String memberName;

    public Member(int memberId, String memberName) {
        this.memberId = memberId;
        this.memberName = memberName;
    }

    public int getMemberId() {
        return memberId;
    }

    public void setMemberId(int memberId) {
        this.memberId = memberId;
    }

    public String getMemberName() {
        return memberName;
    }

    public void setMemberName(String memberName) {
        this.memberName = memberName;
    }

    @Override
    public String toString() {
        return memberName + " 회원님의 아이디는 " + memberId;
    }
}

```
